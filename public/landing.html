How it works

Four steps, zero complexity

Your app sends one POST. LumiGate handles search, tool execution, file generation, and streams back clean results.

1

Send a message

Your app posts to /v1/chat with provider, model, and messages. Standard OpenAI format — nothing new to learn.

2

Auto-enhance

LumiGate detects search intent, generates multi-language keywords, and queries SearXNG. Fresh context injected automatically.

3

Execute tools

AI outputs tool tags, server intercepts and runs them. Spreadsheets generated, code executed, results collected — all server-side.

4

Clean response

Client receives only clean text, file download links, and status events. Zero tool logic needed in your frontend.

Security

Defense in depth,
zero overhead

Every check is O(1) in-memory. Adding all security layers actually improved throughput in benchmarks — no measurable cost.

Four Auth Modes

Direct Key for server-to-server. HMAC Signature — key signs but never transmits. Ephemeral Token — short-lived bearer tokens. HMAC + Token Combo for maximum mobile security.

PII + Secret Masking

Regex + optional Ollama semantic PII detection. Secrets replaced with [SEC_xxx] placeholders in LLM context, restored only at tool execution. 17 command guard rules block dangerous shell patterns.

Per-project Limits

Each project has independent RPM cap, daily/monthly budget, IP allowlist (up to 50 entries), model allowlist, and anomaly auto-suspend on 5x traffic spike.

Audit Trail + MFA

All security events logged to PocketBase audit_log collection in real time. Dashboard protected by TOTP MFA. Admin actions require re-authentication.
