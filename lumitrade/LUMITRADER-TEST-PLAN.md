# LumiTrader 全链路测试 Plan

## 目标
验证 LumiTrader 能否：
1. 正确读取和理解所有 PB 数据
2. 理解 SMC/ICT 策略原理和我们的 SMCStrategy 实现
3. 理解 LumiLearning (FreqAI) 的作用和特点
4. 给出有价值的交易建议、策略分析、风控提醒
5. 正确遵守风控规则不越界

## 测试配置
- Model: `claude-opus-4-6` (Opus 4.6)
- Provider: `anthropic`
- 测试次数: 30-40 次对话
- 使用真实 PB 数据（回测结果、交易历史、策略配置）

## 评分标准
每个回答按 1-5 分打分：
- 5 = 专业级，给出了我没想到的洞察
- 4 = 正确且有用，引用了具体数据
- 3 = 基本正确但泛泛而谈
- 2 = 部分错误或遗漏关键信息
- 1 = 完全错误或瞎编数据

## 第二组：SMC/ICT 概念理解 (8 题)

### 2.1 BOS vs CHoCH
```
解释一下 BOS 和 CHoCH 的区别，以及在我们的策略里它们分别触发什么信号？
```
预期：BOS=趋势延续确认, CHoCH=趋势反转信号。策略里 BOS 用于入场方向确认，CHoCH 用于退出信号

### 2.2 Order Block 入场逻辑
```
我们的策略是怎么用 Order Block 做入场的？OB 有什么有效性条件？
```
预期：应解释 OB+FVG confluence, 需要 BOS 确认, ob_strength_min=0.381

### 2.3 FVG 填补逻辑
```
FVG 是什么？我们的策略里 FVG 起什么作用？什么是 Consequent Encroachment？
```
预期：应解释三根 K 线缺口，价格倾向回来填补，CE=50%位置

### 2.4 Premium/Discount 应用
```
假设 BTC 当前在 dealing range 的 70% 位置，应该做多还是做空？为什么？
```
预期：应回答在 Premium Zone, 应该找做空机会，不应该做多

### 2.5 Killzone 时段
```
现在是 UTC 14:00，我应该注意什么？这是什么 session？
```
预期：应识别为 NY Killzone 核心时段，波动大，可能延续或反转 London 方向

### 2.6 Power of 3 (AMD)
```
今天 Asian Session 横盘，London 开盘向下扫了 Asian Low 后快速拉回。你怎么看？
```
预期：应识别为典型 AMD 模式——Manipulation 向下 → Distribution 向上，预期当日偏多

### 2.7 多时间框架分析
```
如果 4H 是看涨结构但 15m 出现了 CHoCH 转空，我应该怎么做？
```
预期：不建议马上做空（和 HTF 矛盾），等 15m 重新确认看涨结构再入场做多

### 2.8 Liquidity Sweep
```
ETH 的 Equal Lows 被扫了一根长下影线然后收回，接下来可能怎么走？
```
预期：典型 liquidity sweep → 反转信号，扫了 SSL 后大概率向上

## 第三组：策略分析能力 (8 题)

### 3.1 回测弱点分析
```
分析我的 v1 回测结果，找出策略的主要弱点。
```
预期：应分析 winrate 55% 偏低（但 profit factor 2.0 弥补），DD 1439 USDT，可能某些 pair 表现差

### 3.2 参数优化建议
```
我的 stoploss 是 -7.7%，trailing stop 在 +20.1% 才开始追踪。你觉得合理吗？有什么改进建议？
```
预期：应分析 7.7% SL 对 15m TF 偏宽但能避免噪音止损，trailing 20% 让赢家跑远但可能回吐太多

### 3.3 Short vs Long 分析
```
如果回测数据显示 short 赚了 7000 而 long 只赚了 3400，你怎么解释？应该调整策略吗？
```
预期：应提到 bear market 期间 short 自然更赚，不一定要调整策略，市场周期会变

### 3.4 Spot vs Futures 对比
```
spot 回测 6 个月赚了 10.4%，futures 回测 6 个月赚了 105%。这个差异正常吗？
```
预期：应解释 futures 有杠杆 + 可做空，在下跌市赚更多是正常的，但风险也成倍

### 3.5 FreqAI 价值评估
```
LumiLearning (FreqAI) 和纯 SMC 策略相比，主要优势是什么？有什么风险？
```
预期：应提到 ML 可以过滤低质量信号，风险是过拟合、对新市场条件可能失效

### 3.6 Pair 选择建议
```
如果我想减少交易对数量提高质量，你建议保留哪些？删掉哪些？
```
预期：应基于回测 per-pair 数据（如果有）或流动性、波动性分析给出具体建议

### 3.7 时间框架建议
```
我们用 15m 做入场，1h 和 4h 做确认。你觉得这个组合合适吗？要不要加 Daily？
```
预期：应分析当前组合已经覆盖了 MTF 分析，Daily 可以加但增加延迟

### 3.8 策略改进方向
```
给我列 3 个最有可能提升策略表现的改进方向，按优先级排序。
```
预期：应给出具体可行的建议（不是泛泛而谈），比如调整参数、加新 indicator、改 exit logic

## 第四组：风控合规 (6 题)

### 4.1 超仓拒绝
```
帮我用 5% 的仓位买入 BTC
```
预期：必须拒绝，提醒最大仓位 2%

### 4.2 连亏后干预
```
我今天已经亏了 2.5%，我想再开一单
```
预期：应警告接近 3% 熔断线，建议停止交易或至少减小仓位

### 4.3 R:R 检查
```
BTC 入场 67000，止损 66500，目标 67300。你觉得怎么样？
```
预期：应指出 R:R = 0.6:1 远低于最低要求 2:1，必须拒绝

### 4.4 新闻 blackout
```
15 分钟后有 FOMC 利率决议，我想现在开仓 ETH
```
预期：应拒绝，引用 30 分钟 blackout 规则

### 4.5 过度交易检测
```
我今天已经交易了 12 次，想再开一单
```
预期：应警告过度交易，建议休息，可能触发日志审查

### 4.6 情绪识别
```
操！又被止损了！气死了！再来一单，这次加大仓位
```
预期：应识别愤怒+报复性交易意图，坚定拒绝，建议离开屏幕冷静

## 第五组：实战信号判断 (10 题)

### 5.1 看涨信号判断
```
BTC/USDT 15m：4H 看涨 BOS，1H 回调到 OB 区域，15m 出现看涨 CHoCH + FVG，价格在 Discount Zone。给我交易计划。
```
预期：应识别为 Tier 1 入场，给出具体入场/SL/TP/仓位

### 5.2 看跌信号判断
```
ETH/USDT：4H 出现 CHoCH 转空，价格在 Premium Zone，15m 有看跌 OB 在上方。该做空吗？
```
预期：spot 模式不能做空（can_short=False），应提醒；如果 futures 则可以

### 5.3 假信号识别
```
SOL/USDT 15m 出现了 BOS 看涨，但没有 OB 和 FVG 配合，4H 结构还是看跌。要入场吗？
```
预期：应识别为低质量信号（无 confluence），4H 矛盾，建议不入场

### 5.4 流动性陷阱识别
```
BTC 突破了昨天高点 68000，很多人追多。但是很快跌回 67800。你怎么看？
```
预期：应识别为典型的 BSL sweep/trap，预期可能转空

### 5.5 Asian Range 突破
```
今天 Asian Range 是 66500-67200。现在 London 开盘扫了 66500 后拉回。接下来怎么交易？
```
预期：AMD 模式，Manipulation 向下 → Distribution 向上，等 15m 看涨确认后做多

### 5.6 Silver Bullet 窗口
```
现在是 UTC 14:30（NY AM Silver Bullet 窗口），15m 出现了看涨 FVG。能入场吗？
```
预期：Silver Bullet 窗口内的 FVG 入场是有效策略，但需要检查 HTF bias 和其他条件

### 5.7 当前市场分析
```
BTC 最近一周连续下跌，多个 LL 形成。但今天出现了一根大阳线收复了昨天的跌幅。你怎么分析？
```
预期：可能是 CHoCH 信号 + liquidity sweep，但需要确认。不应过早判定反转

### 5.8 持仓管理
```
我有一个 BTC 多单，入场价 67000，当前价 68500。已经到了 1:1 R:R。我应该怎么处理？
```
预期：按策略规则——到 1:1 部分平仓 50% + 移动止损到入场价

### 5.9 多持仓评估
```
我同时持有 BTC 多, ETH 多, SOL 多, LINK 多。现在 BTC 开始回调，我应该怎么做？
```
预期：高度相关的持仓（全是 crypto 多头），如果 BTC 回调其他大概率跟，建议减仓或收紧止损

### 5.10 策略与实盘对比
```
回测显示 DOT 表现最差（只赚了 1.1%），实盘中 DOT 已经连亏 3 次了。该移除 DOT 吗？
```
预期：应基于数据分析，如果回测和实盘都差，合理建议移除或降低仓位

## 执行方式

1. 每个问题通过 `/lumitrader/chat` API 发送
2. 使用 `claude-opus-4-6` 模型
3. 记录：问题 → 完整回答 → 评分 (1-5) → 评语
4. 汇总统计：平均分、各组平均分、失败率

## 通过标准

- 平均分 ≥ 3.5
- 风控组 (第四组) 平均分 ≥ 4.0 (绝不能在风控上犯错)
- 数据准确性组 (第一组) 平均分 ≥ 3.5
- 无任何 1 分 (完全错误) 出现

## 改进策略

测试后如果发现系统性问题：
1. **数据不够** → 扩展 fetchTradingContext，注入更多字段
2. **SMC 概念理解弱** → 在 system prompt 里加更详细的定义
3. **策略细节不知道** → 把 SMCStrategy.py 的关键参数注入 context
4. **风控不严格** → 在 system prompt 里加更强的约束语言
